Aiming to connect user-made events with user-hosted event spaces, we designed an in-app platform to meld seamlessly within the airbnb experience.

Design Goals

We sought to overlap two user paths in the same subplatform. Deploying the canonical ux design process, we explored novel designs for an in-app platform connecting users to one another, threading user flows seamlessly within the existing airbnb product experience.

Process

Working alongside 4 other designers, I directed the development of the design system and product branding for this experience. We shared all roles for this design process including wireframing, iterative user testing, prototyping, and stakeholder presentations.

User Journey

Bringing together our two personas, a "space-seller" and an "event-thrower," we sought to design a platform where both parties could browse and search for available events or spaces seamlessly.

Research

User research and contextual inquiry indicated this would be an attractive product for both native and platform-naïve users - beginning with user expectations, we developed a prototype driven by intuitive, MVP-led design and incorporating ux best practices.

Prototyping

In low-fidelity prototypes, venue owners are able to create a profile for their venue, include relevant venue information, and share openly on the platform. Event hosts are similarly able to profile their event, include event details, and share publicly. Both parties are able to reach out to one another directly.

Usability Testing

Usability testing identified key user pain points. These included issues with the general card layouts, specifically confusion about CTA placement or content, and the use of language that was uncommon or unhelpful.

Priortizing Features

Data from user testing and user interviews, in conjunction with stakeholder consideratons, helped us to funnel our core features into a cohesive product solution. Addressing our users' primary pain points, we focused on these core issues for our final prototype.

Hi-Fi Prototyping

Functional user experience running on invision. Layouts and language were optimized according to best practices, as directed by user feedback. usability testing showed that with this prototype, most users noted the flow was indistinguishable from the existing airbnb experience.
